### 问题(2)：GSG 频谱权重可视化

> **审稿意见原文**：在不同层级的全局频谱门控块中，文中引入可学习的频谱权重Wi以对不同频段进行自适应抑制。尽管文中指出该设计能够有效过滤扩散过程引入的高频噪声，但目前仍缺乏相应的可视化验证。建议文中进一步展示不同门控块所关注的频谱范围或响应分布，以更直观地说明各层频谱门控机制的实际作用及其差异。

**类型**：可视化实验  
**优先级**：🟡 中  
**耗时**：1-2天

**具体操作**：

1. **提取频谱权重**：
   - 加载训练好的模型权重
   - 从 `FeatureAlignAndFuse` → `DynamicConditionFusion` → `FFParser` 中提取 `self.complex_weight`（4个尺度）
   - 每个权重形状为 `(C, H, W//2+1, 2)`，转为复数后取幅值

2. **可视化方式**（建议两种）：
   - **方式A - 频谱响应热力图**：对每个尺度的 W_i 取通道平均，得到 2D 频域响应图，用 matplotlib 的 `imshow` 展示，低频在中心
   - **方式B - 径向频率分析**：将频谱权重按频率距离（到DC分量的距离）分组，画出"频率 vs 平均权重幅值"曲线，展示高频被抑制的趋势

3. **代码实现思路**：
```python
# 伪代码
model = load_model(checkpoint_path)
for i, fuse in enumerate([model.backbone.fuse1, model.backbone.fuse2, model.backbone.fuse3, model.backbone.fuse4]):
    W = fuse.dynamic_fusion.ffparser.complex_weight  # (C, H, W, 2)
    W_complex = torch.view_as_complex(W)  # (C, H, W)
    magnitude = W_complex.abs().mean(dim=0)  # (H, W) 通道平均
    # 可视化 magnitude
```

4. **预期结论**：低层GSG（fuse1/fuse2）保留更多高频（边缘信息相关），高层GSG（fuse3/fuse4）更强地抑制高频噪声，不同层关注不同频段。

### 问题(3)：补充推理流程示意图

> **审稿意见原文**：文中图 2 已较为直观地展示了 GLC-Diff 在训练阶段的整体流程，包括前向扩散、条件控制机制以及反向去噪过程。鉴于本文在推理阶段是以高斯噪声xT为起点，并结合篡改图像提取的条件特征c逐步完成掩码重建，建议文中进一步补充一个更清晰的推理流程示意图，以便读者直观理解测试阶段与训练阶段在流程上的联系与区别。

**类型**：图表补充  
**优先级**：🟡 中  
**耗时**：1天

**具体操作**：

用绘图工具（PPT/Visio/TikZ）绘制推理阶段流程图，核心要素：

```
推理流程：
┌─────────────────────────────────────────────────┐
│  x_T ~ N(0, I)  (纯高斯噪声)                      │
│       ↓                                          │
│  ┌──────────────────────────────────────┐        │
│  │  循环 t = T, T-1, ..., 1：            │        │
│  │    1. Encoder(x_t, t) → noise_feats  │        │
│  │    2. PVT(I) + GSG(noise_feats)      │        │
│  │       → F_c, F_g                     │        │
│  │    3. LEG(F_c) → F_l                 │        │
│  │    4. DGLF(F_c, F_g, F_l) → c       │        │
│  │    5. Denoise: x_{t-1} = f(x_t, c)  │        │
│  └──────────────────────────────────────┘        │
│       ↓                                          │
│  x_0 = 预测掩码                                   │
└─────────────────────────────────────────────────┘
```

**与训练阶段的区别需要标注**：
- 训练：x_t 由真实掩码加噪得到；推理：x_T 从纯噪声开始
- 训练：单步预测（随机采样一个 t）；推理：T 步迭代
- 训练：有 GT 监督；推理：无 GT

> **审稿意见原文**：本文骨干网络对比部分中，仅对比了ResNet以及PVT的不同版本，无法排除性能提升是否与Vision Transformer本身有关。要得出“PVT所提供的层次化、多尺度表征”与GLC-Diff的设计理念高度一致的结论，建议补充尺度不变ViT及其变体（如DeiT等）的性能对比以及其他多尺度ViT变体（如Swin Transformer等）的性能对比。

   - DeiT 是单尺度输出，需要在接入 GLC-Diff 时进行特征分层（如取不同深度的 block 输出）
