## 〇、统一对比方法选取逻辑

### 核心原则

为确保全文各补充实验的对比方法一致且有代表性，按照论文相关工作中的分类逻辑（CNN / Transformer / CNN-Transformer混合 / 扩散模型），从**表2已有方法**中各类选取代表，形成统一的对比方法池。

### 统一对比方法池

| 方法 | 架构类型 | 开源地址 | 选取理由 |
|------|---------|---------|---------|
| **PSCC-Net** | CNN | https://github.com/proteus1991/PSCC-Net | 表2中CNN代表，轻量级 |
| **MVSS-Net** | CNN | https://github.com/dong03/MVSS-Net | 表2中CNN代表，经典多尺度 |
| **ObjectFormer** | Transformer | https://github.com/jiangshuguang/ObjectFormer (需确认) | 表2中Transformer代表 |
| **NCL-IML** | Transformer | https://github.com/knightyxp/NCL-IML | 表2中Transformer代表，有噪声对比学习 |
| **HDF-Net** | CNN-Transformer混合 | https://github.com/ruidonghan/HDF-Net | 表2中混合架构代表，次优baseline |
| **GLC-Diff (本文)** | 扩散模型 | 已有 | 本文方法 |

### 在各实验中的使用规则

| 补充实验 | 从方法池中选取 | 备注 |
|---------|--------------|------|
| 问题(18) Part A：实验设置三（AI跨域泛化） | **全部5个** + GLC-Diff | 统一27K预训练，直接在AI数据集测试 |
| 问题(4)：实验设置二补方法 | **HDF-Net + NCL-IML + ObjectFormer** | 表4已有ManTraNet/MVSS-Net，补3个 |
| 问题(5)+(19)：鲁棒性分析 | **全部5个** + GLC-Diff | 确保与表2一致 |

### 为什么不选 PIM-Net / ManTraNet / TruFor？

| 方法 | 是否入选 | 原因 |
|------|---------|------|
| PIM-Net | ❌ 不选 | 不在表2中，且开源地址不明确，增加风险 |
| ManTraNet | ❌ 不额外选 | 表4中已有结果（0.596），鲁棒性实验中可保留 |
| TruFor | ❌ 不选 | 训练流程特殊（需自监督预训练指纹），难以公平对比 |
| CAT-Net / SPAN | ❌ 不选 | 不在表2中，且开源复现困难 |

### 最终决策：以表2为准

> **所有补充实验的对比方法 = 表2中的5个方法（PSCC-Net、MVSS-Net、ObjectFormer、NCL-IML、HDF-Net）**
> 
> 好处：
> 1. 论文全文对比方法完全统一，审稿人不会质疑"为什么某些实验选了不同方法"
> 2. 这5个方法均已在表2中有结果，说明你已具备其训练/推理能力
> 3. 覆盖了 CNN(2个) + Transformer(2个) + 混合(1个) + 扩散(本文) 四种架构

## 一、问题(18)：AI生成篡改跨域泛化实验

### 原始意见

> 本文对比实验部分中，"实验设置1"的训练与评估都针对传统篡改类型，传统篡改与AI生成修复篡改之间的跨域泛化能力验证不充分，建议在"实验设置1"的评估数据集中加入AI生成类的篡改图像数据集，比如CocoGlide、FF++、AutoSplice等，进一步验证模型的泛化性。"实验设置2"中，仅使用了一种基于Diffusion的方法进行性能对比（InpDiffusion），建议对比更多近期的其他基于Diffusion的伪造定位工作。

### 拟解决方案

**新增"实验设置三"**：挑选 5 个方法，统一在 CASIA v2 上训练，然后在AI生成篡改数据集（CocoGlide/AutoSplice）上直接测试，验证传统篡改→AI篡改的跨域泛化能力。

### 需要下载的数据集

| 数据集 | 规模 | 篡改方式 | 下载地址 |
|--------|------|---------|---------|
| CocoGlide | 512张 | GLIDE文本引导修复 | https://github.com/grip-unina/TruFor （TruFor仓库内附带） |
| AutoSplice | 3621张 | DALL-E 2辅助拼接 | https://github.com/shanface33/AutoSplice_Dataset |

### 参与对比的方法（与统一方法池一致）

| 方法 | 架构类型 | 开源地址 | 需要做什么 |
|------|---------|---------|-----------|
| **PSCC-Net** | CNN | https://github.com/proteus1991/PSCC-Net | 在CASIA v2上训练 |
| **MVSS-Net** | CNN | https://github.com/dong03/MVSS-Net | ✅ 已有CASIAv2权重可直接用 |
| **ObjectFormer** | Transformer | 需确认开源地址 | 在CASIA v2上训练 |
| **NCL-IML** | Transformer | https://github.com/Knightzjz/NCL-IML | ⚠️ 无训练代码，用官方预训练权重推理 |
| **HDF-Net** | CNN-Transformer混合 | https://github.com/ruidonghan/HDF-Net | 在CASIA v2上训练 |
| **GLC-Diff (本文)** | 扩散模型 | 已有 | 使用已有的CASIA v2微调权重 |

### 训练设置

- **训练数据**：CASIA v2（5123张，传统篡改类型：拼接+移除）
- **训练策略**：各方法用各自默认超参数，在CASIA v2上训练
- **不在AI数据集上做任何微调**：训练仅使用传统篡改，测试直接在AI生成篡改上跑
- **输入分辨率**：各方法按自身默认分辨率（PSCC-Net 256, HDF-Net 256, MVSS-Net 512, NCL-IML 512, GLC-Diff 352）

> **为什么用CASIA v2而不是27K？**
> 1. CASIA v2是公认benchmark，更有说服力
> 2. 审稿人原意是验证"传统篡改训练→AI篡改测试"的跨域泛化，CASIA v2正好是纯传统篡改
> 3. MVSS-Net已有现成CASIAv2权重，减少工作量
> 4. 数据量小（5123张），各方法训练快，1-2天即可完成

### 测试设置

- **测试数据**：CocoGlide 全部 512 张 + AutoSplice 全部 3621 张
- **评价指标**：F1、AUC
- **无阈值搜索**：F1 采用固定阈值 0.5 或最优阈值（与表3一致）

### 论文中新增内容

#### 开源现状调研（2026.6 最新）

| 方法 | 论文/会议 | GitHub 地址 | 代码状态 | 权重状态 |

| Training-free方法 | AAAI 2025 | 未找到 | ❌ | ❌ |

#### 拟解决方案

针对开源现状，采取如下策略：

**方案一（推荐）：直接使用 DiffForensics 已释放的 prediction maps**

DiffForensics 虽未开源代码，但作者在百度网盘上释放了各数据集的 pixel-level 预测结果图。如果其释放的预测包含 DID 数据集结果（或其他可对比数据集），我们可以：
1. 下载 DiffForensics 的 prediction maps（百度网盘，提取码: hgzw）
2. 直接用这些预测图计算 AUC/F1 指标
3. 加入表4

**方案二：使用 DiffIML 代码复现**

DiffIML (`scu-zjz/DiffIML`) 有完整代码，基于 IMDLBenCo 框架：
1. 下载代码：`git clone https://github.com/scu-zjz/DiffIML`
2. 在 Inpainting32K 上训练（或使用其默认训练方案）
3. 在 DID 数据集上测试

**方案三：Response Letter 中说明**

如果以上方案均无法获取有效结果，在 response letter 中说明：
> "目前基于扩散模型的IML方法数量十分有限且多数尚未开源代码（如DiffForensics仅释放了预测结果图但未公开训练代码），我们尽最大努力对比了所有可获取的方法。"

1. **DiffForensics prediction maps 获取**：
   - 访问 `https://github.com/ZeqinYu/DiffForensics` 中 README 的网盘链接

   - 如训练太复杂/耗时过长，可尝试其预训练权重直接测试

   - DiffForensics：引 CVPR 2024 论文，注明"结果取自作者公开的预测图"

## 七、问题(2)：GSG频谱权重可视化

### 原始意见

> 在不同层级的全局频谱门控块中，文中引入可学习的频谱权重Wi以对不同频段进行自适应抑制。尽管文中指出该设计能够有效过滤扩散过程引入的高频噪声，但目前仍缺乏相应的可视化验证。建议文中进一步展示不同门控块所关注的频谱范围或响应分布，以更直观地说明各层频谱门控机制的实际作用及其差异。

### 拟解决方案

从训练好的模型中提取4个尺度的FFParser频谱权重Wi，做两种可视化。

### 需要做的事

1. **提取权重**：加载 GLC-Diff 的checkpoint，提取 `backbone.fuse1~4.dynamic_fusion.ffparser.complex_weight`
2. **可视化方式A — 2D频谱响应热力图**：
   - 将 complex_weight (C, H, W, 2) 转为复数取幅值
   - 通道取平均得到 (H, W) 的2D频谱响应图
   - 4个尺度分别画一张热力图（低频在中心）
3. **可视化方式B — 径向频率曲线**：
   - 将频谱权重按到DC分量的距离分组
   - 画"频率距离 vs 平均权重幅值"曲线
   - 4个尺度画在同一张图上对比

### 论文中新增内容

**新增一张图**（如图9）：包含两行
- 第一行：4个尺度的2D频谱响应热力图
- 第二行：径向频率分析曲线（4条线对比）

**配文字分析**：
> "如图X所示，低层GSG（fuse1/fuse2）对高频分量保留相对较多，与边缘信息的保持一致；高层GSG（fuse3/fuse4）对高频成分的抑制明显更强，有效过滤了扩散过程引入的高频噪声。这一结果验证了GSG能够根据不同尺度的语义需求，自适应地学习差异化的频域滤波策略。"

### 代码实现

```python
import torch
import matplotlib.pyplot as plt
import numpy as np

# 加载模型
checkpoint = torch.load('path/to/best_model.pth', map_location='cpu')
model_state = checkpoint['model']  # 根据实际key调整

# 提取4个尺度的频谱权重
fuse_names = [
    'backbone.fuse1.dynamic_fusion.ffparser.complex_weight',
    'backbone.fuse2.dynamic_fusion.ffparser.complex_weight',
    'backbone.fuse3.dynamic_fusion.ffparser.complex_weight',
    'backbone.fuse4.dynamic_fusion.ffparser.complex_weight',
]

fig, axes = plt.subplots(1, 4, figsize=(16, 4))
for i, name in enumerate(fuse_names):
    W = model_state[name]  # (C, H, W, 2)
    W_complex = torch.view_as_complex(W)  # (C, H, W)
    magnitude = W_complex.abs().mean(dim=0).numpy()  # (H, W)
    axes[i].imshow(magnitude, cmap='hot')
    axes[i].set_title(f'GSG Scale {i+1}')
    axes[i].axis('off')
plt.savefig('gsg_spectral_weights.pdf', bbox_inches='tight')
```
